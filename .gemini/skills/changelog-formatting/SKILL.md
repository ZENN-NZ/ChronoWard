# Changelog Formatting Standards

Always format new entries in `CHANGELOG.md` using the exact project conventions:

## [X.Y.Z] - YYYY-MM-DD

### Fixed
- **Category:** Concise bold title — detailed description of fix
- Bullet point referencing file paths when appropriate

### Changed
- `file/path.rs` — specific description of refactored logic or configuration update

### Security
- Description of security enhancements, keychain logic fixes, or audit checks

### Added
- Summary of new user-facing features or options
